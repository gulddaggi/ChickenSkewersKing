# 게임 방향 정리

## 핵심 플레이 감각
플레이어는 제한된 시간 안에 닭꼬치를 정확하게 만들고 조리해서
높은 점수를 노리는 푸드트럭 운영자를 플레이한다.

이 게임은 다음 감각을 줘야 한다.

- 시간이 줄어드는 긴박함
- 빠르고 정확한 마이크로 컨트롤의 손맛
- 실수를 줄였을 때의 만족감
- 고득점과 랭킹으로 이어지는 성취감

## 현재 핵심 설계 요소
- 짧은 세션 기반의 아케이드 플레이
- 제한 시간 내 최대 점수 달성
- 레시피 정확도와 실수 횟수에 따른 점수 차등
- 주문 대응 속도와 완성도에 따른 평판 변화
- 평판에 따른 주문 빈도 변화
- 결과 화면에서 점수와 랭킹 확인
- 튜토리얼 또는 설명 구간 필요

## 핵심 게임 루프
플레이의 기본 흐름은 아래와 같다.

1. 주문 확인
2. 재료 선택 및 꼬치 제작
3. 조리 진행
4. 마무리 및 제출
5. 점수와 평판 반영
6. 다음 주문 처리

## 중간 시스템 목표
게임잼 버전에서 본게임으로 확장하기 위해,
다음 시스템을 우선적으로 정리한다.

### 1. 전체 게임 흐름
- 타이틀
- 닉네임 입력
- 튜토리얼 또는 설명
- 플레이
- 결과
- 랭킹

### 2. 주문 흐름
- 주문 생성
- 주문 대기
- 제작 중
- 조리 중
- 제출
- 성공 / 실패 / 시간 초과

### 3. 점수 시스템
점수는 단순 합산이 아니라,
아래 요소를 기준으로 계산하는 방향을 우선한다.

- 재료 정확도
- 입력 성공도
- 조리 완성도
- 제출 성공 여부
- 연속 성공 보너스 여부
- 팁 또는 추가 보상

### 4. 평판 시스템
평판은 단순 점수와 분리된 보조 난이도 축이다.

- 성공적으로 주문을 처리하면 평판 상승
- 시간 초과나 실패 시 평판 하락
- 평판이 높을수록 주문 빈도 또는 난이도 상승
- 높은 리스크를 감당하면 더 높은 점수 기회 제공

## 확장 방향
향후 게임을 확장할 때는 아래 순서를 우선한다.

1. 게임 상태 전이 정리
2. 주문/점수/평판 로직 분리
3. 튜토리얼 흐름 추가
4. 결과 및 랭킹 흐름 정리
5. 스토리라인과 연출 텍스트 추가
6. 이후 필요한 경우 씬 연출 보강

## 스토리라인 추가 원칙
스토리라인은 처음부터 큰 컷신 중심으로 가지 않는다.
먼저 아래처럼 가벼운 방식으로 넣는다.

- 하루 단위 진행
- 짧은 인트로 텍스트
- 손님 반응 문구
- 결과 요약 문구
- 특정 점수/평판 도달 시 해금되는 대사

즉, 초기 스토리라인은
**게임 흐름을 방해하지 않고 플레이의 맥락을 강화하는 수준**으로 시작한다.

## 구현 시 주의
- 큰 씬 개편보다 코드 구조 정리를 먼저 한다.
- 게임 루프와 상태 전이를 코드에서 명확히 만든다.
- UI 연출보다 실제 플레이 흐름이 먼저다.
- 테스트 가능한 계산 로직은 가능한 한 순수 C# 클래스로 분리한다.
